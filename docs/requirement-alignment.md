# 工业数据可视化平台原型需求对齐

## 目标定义

本项目面向工业数据可视化场景，建设一个专注于以下能力的平台原型：

- 2D 组态大屏全生命周期管理
- 3D 场景项目全生命周期管理
- 数据源与数据集管理
- 用户、项目、系统管理
- 基于低代码拖拽方式的 2D/3D 组态设计
- 基于 MySQL 数据源接入与 RBAC 权限体系的管理底座

## 模块拆解

### 1. 大屏可视化

目标功能：

- 项目列表展示
- 新建大屏
- 大屏编辑
- 大屏预览
- 大屏发布
- 项目克隆
- 多条件搜索筛选

当前阶段落地：

- `大屏项目` 列表页：实现筛选区、状态统计、项目卡片、预览/编辑/发布/克隆操作入口
- `2D 编辑器` 页面：实现组件库、画布、图层列表、属性面板的低代码原型

后续补齐：

- 真正的拖拽编排、组件物料协议、画布持久化、发布版本管理

### 2. 三维可视化

目标功能：

- 三维项目列表
- 新建三维场景
- 3D 模型导入
- 画布编辑
- 场景预览
- 场景发布

当前阶段落地：

- `三维场景` 列表页：实现筛选、状态统计、模型数量展示、预览/编辑/发布
- `3D 编辑器` 页面：实现模型资源区、场景树、属性面板、视口工作台原型

后续补齐：

- Three.js 场景引擎、GLTF/FBX 导入、材质和动画配置、设备点位绑定

### 3. 数据管理

目标功能：

- 数据源创建
- 连接配置
- 连通性测试
- 编辑删除
- 基于数据源生成和管理数据集

当前阶段落地：

- `数据源管理` 页面：实现数据源列表、类型筛选、连接状态、测试连接入口
- `数据集管理` 页面：实现数据集列表、来源数据源、刷新策略和字段信息展示

后续补齐：

- 数据源表单、MySQL 真实连通性测试、SQL 建模、字段映射和查询预览

### 4. 用户管理

目标功能：

- 账号检索
- 新增账号
- 资料编辑
- 密码重置
- 账号删除

当前阶段落地：

- `用户管理` 页面：实现搜索、状态过滤、账号表格、重置密码和编辑操作入口

后续补齐：

- 角色配置、部门组织、登录审计、用户详情和密码策略

### 5. 项目管理

目标功能：

- 全局项目统一管理
- 项目检索
- 新建项目分组
- 项目信息编辑
- 无效项目删除

当前阶段落地：

- `项目管理` 页面：实现分组、类型、负责人和状态的统一视图

后续补齐：

- 分组树、跨项目权限绑定、归档/回收机制

## RBAC 对齐

题目要求包含 RBAC 权限体系，当前阶段通过 `系统设置` 页面先落地权限模型说明与角色矩阵展示，后续补齐：

- 用户-角色绑定
- 角色-菜单权限绑定
- 角色-项目权限绑定
- 数据权限范围控制

## 第一阶段开发边界

本阶段聚焦前端原型和页面骨架，不引入后端服务，统一使用模拟数据驱动：

- 完成信息架构、导航和模块页
- 完成 2D/3D 编辑器工作台原型
- 完成数据管理、用户管理、项目管理基础列表
- 为后续接入 MySQL 和权限系统预留数据结构

## 下一阶段建议

1. 引入 Pinia 统一管理项目、画布和用户态
2. 优先完成 2D 编辑器拖拽与画布持久化
3. 接入 MySQL 数据源表单与后端测试接口
4. 接入 Three.js 完成 3D 场景视口
5. 增加 RBAC 角色管理与鉴权守卫
